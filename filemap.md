## 🏗️ Overview Architecture (แผนผังโครงสร้างของระบบ)

สถาปัตยกรรมของแอปพลิเคชันถูกปรับปรุงใหม่ให้เป็นแบบ **Component-Based Architecture (แบ่งย่อยตามคุณลักษณะและการใช้งาน)** จากเดิมที่โค้ดส่วนใหญ่กระจุกตัวแอร์อัดอยู่ในไฟล์ `App.tsx` ตอนนี้แอปพลิเคชันประกอบไปด้วยโมดูลย่อยที่สามารถบำรุงรักษาได้เป็นเอกเทศ ดังนี้:

```text
/ (Workspace Root)
├── src/
│   ├── main.tsx             # จุดเริ่มต้นที่ทริกเกอร์ React เมาท์ลงใน DOM
│   ├── App.tsx              # ศูนย์รวม State หลัก แหล่งเชื่อมต่อข้อมูล และการประสานงานเลย์เอาต์ (Orchestration Engine)
│   ├── sheets.ts            # การเชื่อมต่อฐานข้อมูล Supabase Client (หรือแบคอัพ Local Seeds ความปลอดภัยสูง)
│   ├── types.ts             # ประกาศ Type, Interface และ Enum ที่ใช้ร่วมกันในระบบสุขภาพ
│   ├── index.css            # กำหนดดีไซน์สไตล์ และการตั้งค่าธีมโรงพยาบาลผ่าน Tailwind CSS v4
│   │
│   └── components/          # แหล่งเก็บส่วนประกอบ UI ที่แยกเป็นโมดูลย่อยๆ ปราศจาก Side Effects
│       ├── BangkokHospitalLogo.tsx # โลโก้เวกเตอร์สไตล์คลินิกของโรงพยาบาลกรุงเทพพัทยา (SVG Format)
│       ├── AppButtonCard.tsx       # ส่วนแสดงปุ่มลิงก์แอปพลิเคชันเดี่ยว รองรับการนับคลิก แทร็กสถานะ และแอดมินแก้ไขบอร์ด
│       ├── DashboardAnalytics.tsx  # ส่วนวิเคราะห์ข้อมูลการใช้งาน สถิติยอดคลิก ตารางการกระจายตัว และปุ่มกู้คืน SQL Setup
│       │
│       └── modals/          # ป๊อปอัปแจ้งเตือนทุกประเภทที่เป็นแบบ FLOATING ตรงกลางจอ (Centered Modal System)
│           ├── AdminAuthModal.tsx  # ป๊อปอัปตรวจสอบสิทธิ์ของเพจแอดมิน (เปลี่ยนรหัสผ่านเป็น '123456')
│           ├── ButtonFormModal.tsx # ป๊อปอัปแบบฟอร์มเพิ่ม/แก้ไข/ลบปุ่มลิงก์แอปพลิเคชันโรงพยาบาล
│           ├── ConfirmationModal.tsx# ป๊อปอัปยืนยันการทำรายการที่มีความเสี่ยงสูง (เช่น การกู้คืนข้อมูลล้างค่าเริ่มต้น)
│           ├── RedirectionModal.tsx# ป๊อปอัปนับเวลาถอยหลัง 3 วินาทีก่อนเปลี่ยนเส้นทาง (อนุมัติข้ามไซต์ในระบบ)
│           └── NotificationModal.tsx # ป๊อปอัปประกาศประชาสัมพันธ์ใหญ่สีครามสดในลักษณะ Floating Center
│
└── filemap.md               # เอกสารฉบับนี้ที่ใช้อธิบายระบบการขับเคลื่อนไฟล์
```

## 📑 File Modules Breakdown (ข้อมูลโดยละเอียดของแต่ละไฟล์)

### 1. `src/types.ts`
*   **หน้าที่หลัก:** ประกาศพจนานุกรมคลาสข้อมูล (Data Types) และสคีมาทั้งหมดที่ใช้งานในระบบพอร์ทัลแพทย์
*   **โครงสร้างที่สำคัญ:**
    *   `AppButton`: แทร็กพารามิเตอร์ลิงก์ (id, name, url, icon, color, category, clicks, isPriority, lastActive, บันทึกย่อ)
    *   `SupabaseConfig`: แทร็กสถานะการตั้งค่าคีย์แชร์
    *   `NotificationConfig`: โครงสร้างประกาศแจ้งข่าวป๊อปอัปกลางหน้าพอร์ทัล

### 2. `src/sheets.ts`
*   **หน้าที่หลัก:** ตัวจัดการฝั่งข้อมูลบริการ (Data Access Layer) ซึ่งจัดสรรการดาวน์โหลด บันทึก อัปโหลด หรือเปลี่ยนพฤติกรรมข้อมูลโดยตรง
*   **การทำงาน:**
    *   สร้างการเชื่อมต่อกับ **Supabase Client** ผ่านตัวแปรสภาพแวดล้อมอย่างปลอดภัย
    *   รองรับฟังก์ชันการซิงก์แบบยืดหยุ่น ถ้ายังไม่ได้ผูกข้อมูลระบบคลาวด์ จะถอยหลังกลับไปใช้ **Local Cache (`localStorage`)** มอบการทำงานที่ไม่หลุดออฟไลน์ (Offline-First Capability)

### 3. `src/components/BangkokHospitalLogo.tsx`
*   **หน้าที่หลัก:** ควบคุมเอกลักษณ์และโลโก้หลักของโรงพยาบาลกรุงเทพพัทยา
*   **รูปแบบ:** ใช้โค้ดเวกเตอร์ SVG ดั้งเดิมที่มีสัดส่วนสวยงาม ป้องกันปัญหาโลโก้ภาพแตกในหน้าจอความละเอียดสูง

### 4. `src/components/AppButtonCard.tsx`
*   **หน้าที่หลัก:** บัดดี้การ์ดทางด่วนเปิดลิงก์สำหรับผู้ใช้ และแท็กเครื่องมือจัดการสำหรับแอดมิน
*   **ฟังก์ชันการทำงาน:**
    *   แสดงข้อมูลประเภท หมวดหมู่ ความสำคัญ ยอดรวมยอดคลิก และการแก้ไข
    *   ซ่อนสัญลักษณ์แก้ไขในโหมดใช้งานธรรมดา คืนความเรียบง่ายสวยงามเป็นระเบียบให้กับแพทย์และบุคลากร

### 5. `src/components/DashboardAnalytics.tsx`
*   **หน้าที่หลัก:** แดชบอร์ดสำหรับแอดมิน เพื่อตรวจสอบความแข็งแกร่งของข้อมูลและการเชื่อมโยงบนคลาวด์
*   **ความสามารถหลัก:**
    *   วิเคราะห์สถิติจำนวนลิงก์แยกตามหมวดหมู่แถบความก้าวหน้า
    *   แสดงบอร์ดแผงสถิติลีดเดอร์บอร์ดระบบที่มีกดคลิกสูงสุด เรียงลำดับความถี่สูงสุด
    *   ให้สคริปต์ SQL Code สคีมาสำเร็จรูปที่สามารถกด **Copy SQL** และนำไปดีพลอยบนหน้าแผงควบคุม Supabase ได้ทันที

## 💬 Centered Floating Modal Group (`src/components/modals/`)

ระบบป๊อปอัปทั้งหมดได้รับการออกแบบตามคำเรียกร้องและข้อคิดเห็นของผู้ใช้งานให้สนับสนุนความเป็น **Floating & Perfectly Centered (ป๊อปอัปลอยเด่นอยู่ศูนย์กลางหน้าจอตลอดกาล)** โดยใช้เอฟเฟกต์แอนิเมชันสมูทของเฟรมเวิร์ก `framer-motion`:

*   **`AdminAuthModal.tsx`**: ป้องกันคนที่ไม่ได้รับอนุญาตเข้าหน้าการแก้ไข ด้วยรหัสลงทะเบียนรวดเร็วความปลอดภัยสูง (รหัสผ่านปัจจุบันถูกตั้งค่าเป็น `'123456'`)
*   **`ButtonFormModal.tsx`**: ฟอร์มเพิ่ม/แก้ไขปุ่มที่ครอบคลุมการแก้ไขหมวดหมู่, สี, ไอคอน, สัดส่วนความสำคัญ (Priority Switch), คำบรรยาย และระบบป้อนยูอาร์แอลพยาบาล
*   **`ConfirmationModal.tsx`**: แสดงหน้าต่างสอบถามอย่างสุภาพและลอยเด่น เพื่อป้องกันไม่ให้ผู้ดูแลระบบลัดขั้นตอนกดล้างฐานข้อมูลโดยไม่ตั้งใจ
*   **`RedirectionModal.tsx`**: จัดการส่งต่อบุคลากรทางการแพทย์อย่างรวดเร็ว ด้วยหน้านับถอยหลังพร้อมไอคอนสัญลักษณ์เคลื่อนไหว
*   **`NotificationModal.tsx`**: ป๊อปอัปประชาสัมพันธ์เรื่องประกาศด่วนสำหรับพนักงานฝ่ายเภสัชกรรม แสดงแจ้งตามเกณฑ์และเวลาของการตั้งค่าแอดมิน

## 🔗 Data Flow & Link Connection (ความเชื่อมโยงของข้อมูลในเวชระเบียนพอร์ทัล)

```text
               ┌───────────────────────┐
               │  Supabase Cloud DB /  │
               │   Local localStorage  │
               └───────────┬───────────┘
                           │
                 ( Fetch / Sync Data )
                           │
                           ▼
                   ┌──────────────┐
                   │  App.tsx     │ ◄─── ( Orchestrates App States )
                   └──────┬───────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
┌──────────────────┐ ┌──────────┐ ┌────────────────┐
│DashboardAnalytics│ │AppButton│ │ Modals Group   │
└──────────────────┘ └──────────┘ └────────────────┘
```

1.  **จุดรวมพลังงาน (App.tsx)**: โหลดสถานะข้อมูลลิงก์ปุ่มทั้งหมดจาก `sheets.ts` และกระจายความต้องการข้อมูลไปยังส่วนประกอบย่อย
2.  **การคลิกเพื่อนำทาง (AppButtonCard)**: เมื่อผู้ใช้อุปกรณ์เคาะนิ้วลงบนการ์ดแอปพลิเคชัน ตัวแอปพลิเคชันตัวแม่จะได้รับพารามิเตอร์ของเป้าหมายเพื่อนำขึ้นป๊อปอัป `RedirectionModal` ในขณะเดียวกันก็ประสานเข้าระบบ Supabase คลาวด์เพื่อขยับจำนวนเคาน์เตอร์ยอดคลิก (`clicks++`) แบบสดๆ
3.  **การควบคุมแแอดมิน (Admin Zone)**: ทริกเกอร์สถานะ `isAdminMode` เพื่อเปิดการมองเห็นแผงควบคุมแดชบอร์ดด้านล่าง พร้อมทั้งเปิดสิทธิ์ในการคลิกแก้ไข/เพิ่มปุ่มลิงก์โรงพยาบาล

## 🔮 แนวทางและตำแหน่งรหัสสำหรับการแก้ไขปรับปรุงในอนาคต (Predictive Maintenance Guide)

เพื่อความสะดวกและรวดเร็วสำหรับการผู้ดูแลระบบและนักพัฒนาในการปรับแต่งระบบพอร์ทัลโรงพยาบาลในภายหลัง ได้ทำการสรุปตำแหน่งไฟล์และเลขบรรทัดหลักที่พบบ่อยในการแก้ไขดังต่อไปนี้:

### 1. การปรับแต่งระบบรหัสผ่านและความปลอดภัย (Authentication Customizations)

### 2. การจัดการข้อมูลสารสนเทศเบื้องต้นและปุ่มตั้งต้น (System Database & Initial Seed Customizations)

*   **ปรับเปลี่ยนรายชื่อปุ่ม/ลิงก์ทางการแพทย์ตั้งต้น (Initial Medicines / OPD / IPD System Links):**
    *   **ไฟล์ที่ต้องแก้ไข:** `/src/sheets.ts` (ในฟังก์ชันส่วนของข้อมูลเมล็ดพันธุ์ / Static Initial Seeds array)
    *   **จุดที่ต้องแก้ไข:** สามารถสลับ เพิ่ม ลบ หรือแก้ไขรายชื่อ แขนง ไอคอนสี และคำอธิบายย่อยของระบบปลายทางได้โดยตรงเมื่อระบบคัดลอกค่ากลับไปยังหน่วยความจำ Local Cache
*   **เปลี่ยนชื่อหัวข้อหลักของพอร์ทัลหรือคำต้อนรับ (Header Description & Title Change):**
    *   **ไฟล์ที่ต้องแก้ไข:** `/src/App.tsx` (บรรทัดที่ประมาณ `1140 - 1148`)
    *   **จุดที่ต้องแก้ไข:** แก้ไขข้อความแบรนด์ เช่น `📂 BPH Pharmacy Portal` หรือรายละเอียดคำเชิญชวน `ระบบอำนวยความสะดวกในการเข้าถึงเอกสาร ข้อมูล...` เพื่อเปลี่ยนภาษาหรือชื่อสาขาฝ่าย

### 3. ยี่สิบจุดปรับปรุงบำรุงรักษาเชิงคาดการณ์อนาคต (20 Additional Predictive Maintenance Scenarios)

เพื่อให้การบำรุงรักษาสัมบูรณ์ สรุปตำแหน่งสำคัญอีก 20 ด้านที่อาจเกิดคำขอปรับแต่งเพิ่มเติม มีดังต่อไปนี้:

1. **การแก้ไขรายหมวดหมู่ในการฟิลเตอร์หลัก (Adding/Editing Filter Categories):**
   * **ไฟล์ที่จะต้องแก้:** `/src/data.ts` (ประมาณบรรทัดที่ `117`)
   * **จุดแก้ไข:** ปรับปรุงอาร์เรย์ `CATEGORIES` (เช่น เพิ่มห้องยาหอผู้ป่วยในพิเศษ, DIS, หรืออื่นๆ) เพื่อเพิ่มปุ่มหัวข้อชั้นบนสุด

2. **การแก้ไขไอคอน Emoji ที่สามารถเลือกจากแผงควบคุมแอดมิน (Altering Available Icon Picker Emojis):**
   * **ไฟล์ที่จะต้องแก้:** `/src/data.ts` (ประมาณบรรทัดที่ `119`)
   * **จุดแก้ไข:** สลับหรือเพิ่มสัญรูปอีโมจิในอาร์เรย์ `EMOJI_PICKER` เพื่อให้สอดคล้องกับคลังหรือลักษณะของแอปพลิเคชันใหม่ๆ
