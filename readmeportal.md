# สารบัญระบบความปลอดภัยและคู่มือโครงสร้างไฟล์ (AppPortal Development Manifesto)

เอกสารฉบับนี้จัดทำขึ้นเพื่อแนะนำความรับผิดชอบ หน้าที่ความรับผิดชอบ และความสัมพันธ์เชิงการดึงข้อมูล (Imports / Dependency Tree) ของแต่ละไฟล์คอมโพเนนต์ภายในแอปพลิเคชัน **AppPortal Central Access Gate** 

---

## 📂 รายละเอียดความรับผิดชอบของแต่ละไฟล์ (Functional Breakdown per File)

### 1. `metadata.json` (ข้อมูลโครงสร้างของ Applet)
*   **ความรับผิดชอบ (Responsibility):** กำหนดชื่อแอปพลิเคชัน คำอธิบาย (Description) และสิทธิ์การใช้งานของ iFrame (Frame Permissions) เช่น กล้อง ไมโครโฟน รวมถึงขีดความสามารถพิเศษของเซิร์ฟเวอร์ระบบ AI
*   **ความสัมพันธ์ (Relationship):** ถูกประมวลผลโดยตรงจากแพลตฟอร์ม Google AI Studio เพื่อกำหนดข้อมูลแสดงผลหน้าต่าง Preview
*   **การเชื่อมต่อ:** มีชื่อแอปสอดคล้องกับหัวเรื่องแสดงผลใน `src/App.tsx`

### 2. `package.json` (ศูนย์รวมการจัดการเวอร์ชันเครื่องมือและไลบรารี)
*   **ความรับผิดชอบ (Responsibility):** บันทึกรายการโมดูลภายนอกที่จำเป็นในการสร้างแอป สคริปต์รันระบบสำหรับการพัฒนาและทำโปรดักชันบิลด์ (`npm run dev`/`npm run build`)
*   **ความสัมพันธ์ (Relationship):** เป็นผู้ติดตั้งแพ็กเกจที่สำคัญ เช่น:
    *   `react` และ `react-dom` (เวอร์ชัน 19) ที่ใช้รันใน `src/main.tsx`
    *   `motion/react` (เดิมคือ Framer Motion) ที่ใช้ทำแอนิเมชันเคลื่อนไหวลื่นไหลใน `src/App.tsx`
    *   `lucide-react` ไลบรารีไอคอนเวกเตอร์ที่จัดหาไอคอนระบบความปลอดภัยให้กับ `src/App.tsx`

### 3. `src/types.ts` (ไฟล์ระเบียบและสัญญารูปแบบข้อมูล)
*   **ความรับผิดชอบ (Responsibility):** เป็นศูนย์กลางตัวกำหนดประเภทข้อมูลคงที่ (Strict TypeScript interfaces) เพื่อความปลอดภัยและไร้ข้อบกพร่องทางโค้ด
*   **โครงสร้างโมเดลสำคัญ:**
    *   `AppButton`: ระบุกุญแจหลัก สี สเกลปุ่ม พารามิเตอร์ลิงก์ไอคอน และประวัติจำนวนครั้งการกดเข้าลิงก์ภายนอก (`clicks`)
    *   `ButtonSize`: สัญญาระบุขนาดที่อนุญาต (`'small' | 'medium' | 'large' | 'xl'`)
*   **ความสัมพันธ์เชิงการดึงค่า (Downstream Pull):**
    *   ถูก **ดึงข้อมูล (Import)** ไปใช้ใน `src/data.ts` เพื่อสร้างข้อมูลแบบปุ่มตั้งต้น
    *   ถูก **ดึงข้อมูล (Import)** ไปใช้ใน `src/App.tsx` เพื่อตรวจสอบสัญญารันไทม์ภายในเมธอด React hooks และตัวฟิลเตอร์ไอคอน

### 4. `src/data.ts` (คฤหาสน์ของข้อมูลตั้งต้นและสเปกธีมสีแดงสลับน้ำเงิน)
*   **ความรับผิดชอบ (Responsibility):** จัดสรรค่ายาและสารสนเทศตั้งต้นของหน่วยบริการ เช่น **BPH Pharmacy Stock** และ **Prescription Control Board** มีการนิยามตัวแปรค่าคงที่ดังนี้:
    *   `INITIAL_BUTTONS`: อาร์เรย์ของปุ่มแอปที่เก็บลิงก์จริงและข้อมูลเบื้องต้น
    *   `COLOR_PRESETS`: แม่แบบรายละเอียดสีสำหรับปุ่ม โดยเน้นเฉดสี **Navy Blue, Azure Blue, Indigo Blue** และชุดสีแดงนำสายตาอย่าง **Crimson Red, Scarlet Coral** และ **Rose Red**
    *   `CATEGORIES` และ `EMOJI_PICKER` ลิสต์สัญลักษณ์แสดงในหน้าแผงตัวเพิ่มเพื่อลดข้อจำกัดของแอดมิน
*   **ความสัมพันธ์เชิงการดึงค่า (Downstream Pull):**
    *   **ดึงข้อมูลมาจาก** `src/types.ts` เพื่อทำให้ข้อมูลปุ่มทุกตัวสอดรับตามข้อกำหนดความมั่นคง
    *   **ส่งออกตัวแปร** `INITIAL_BUTTONS` และ `COLOR_PRESETS` แก่ `src/App.tsx` เพื่อนำไปใช้งานด้านสเตตภายในตัวเก็บความจำเบราว์เซอร์ (`localStorage`)

### 5. `src/App.tsx` (เครื่องยนต์แอปพลิเคชันหลัก - Core Application Engine)
*   **ความรับผิดชอบ (Responsibility):** ดำเนินหน้าที่และมีผู้ร่วมการควบคุมแบบปฏิสัมพันธ์ (User Interaction Host) ประกอบด้วย:
    *   **ระบบหน้าจอเข้าสู่ระบบ (Medical Login Gateway):** ปกป้องความมั่นคงด้วยไอดีและรหัสผ่านหวงห้ามหลัก (`PharmacyBPH` และ `#Pharm12`) ตกแต่งด้วยดีไซน์ไล่เฉดสีนวัตกรรมน้ำเงินเข้มขลิบแดง
    *   **ระบบยืนยันสิทธิ์แอดมินแบบ 3 ด่าน (3-Step Advanced Verification Modal):** ป้องกันการกดเปิดสวิตช์ผู้ดูแลโดยพละการด้วยชุดรหัสพิเศษที่คีย์ผ่านการไล่ระดับอย่างละเมียดละไม (ขั้นที่ 1: `221008` ➔ ขั้นที่ 2: `25031998` ➔ ขั้นที่ 3: `0891707117a`)
    *   **ระบบจัดการควบคุมปุ่ม (Administrative Controls):** ดำเนินการเพิ่ม ลบ เปลี่ยนสเกล แก้ไขลิงก์ปลายทาง และเปลี่ยนสีด่วนให้แก่แผงควบคุมหลัก
    *   **ระบบจำลองการผ่านลิงก์อย่างคุ้มครองความปลอดภัย (Gateway Routing Logs Console):** จำลองเสมือนจริงการเชื่อมต่อเกตเวย์เพื่อส่งพารามิเตอร์ส่งออกด้วยแอนิเมชันล้ำยุค
*   **ความสัมพันธ์เชิงการดึงค่า (Downstream Pull):**
    *   **ดึงไอคอนจาก** `lucide-react`
    *   **ดึงไลบรารีภาพเคลื่อนไหวจาก** `motion/react`
    *   **ดึงข้อมูลระเบียนและสารสนเทศเริ่มต้นมาจาก** `src/types.ts` และ `src/data.ts`
    *   **ส่งต่อผลบิลด์ความสมบูรณ์** ให้แก่ `src/main.tsx` เพื่อเมานท์เข้าเว็บ

### 6. `src/index.css` (จุดกำเนิดความงามแห่งงานออกแบบ)
*   **ความรับผิดชอบ (Responsibility):** บรรจุฟอนต์พรีเมียมอันเป็นมิตรต่อสายตาระหว่างใช้งาน ได้แก่ ฟอนต์อย่างเป็นทางการของสถานพยาบาล **"TH SarabunPSK"** และ **"Sarabun"** เพื่อความเป็นระเบียบลดความตึงเครียดของสายตา และฟอนต์ **"JetBrains Mono"** สำหรับแสดงเลขระบบปฏิบัติการเชิงสถิติ พร้อมนำเข้าระบบ Tailwind CSS
*   **สัญกรณ์สีระดับพรีเมียม:** ปรับแต่งโครงร่างคลาสย่อยและการไล่สีโปรแกรม (Gradients) ในคอมโพเนนต์ให้เด่นล้ำด้วยโทนสีแบบคลีน มินิมอล เหมาะแก่เวชปฏิบัติภายในโรงพยาบาล ผ่านกรอบสีอ่อนเพื่อให้พนักงานในระบบไม่ต้องเกร็งสายตา

## 🛠️ สรุปขั้นตอนการทำบิลด์ระบบเพื่อการส่งมอบ (Production Build Pipeline)

เมื่อทำการรันโปรแกรมระบบ:
1. การตรวจสอบไทป์อย่างเข้มงวดจะเริ่มขึ้น โดยดึงโครงสร้างของสัญญาจาก `src/types.ts` มาวัดความเข้ากันได้
2. ดีเวลลอปเปอร์สามารถปรับปรุงและตั้งค่าลิงก์ในหน้า UI แอดมินได้ตลอดเวลา ตัวแอปจะสั่นไหวอย่างราบรื่นผ่านเฟรมเวิร์ก `motion/react` พร้อมอัปเดตสถิติคลิกอัตโนมัติ 
3. ข้อมูลทั้งหมดจะถูกเขียนและอ่านต่อเนื่องผ่านเทคโนโลยีการบันทึกสถานะผู้จำหน่ายร่วม (`localStorage` สำหรับข้อมูลเว็บ และ `sessionStorage` สำหรับการล็อกอินของผู้ใช้ในรอบปฏิบัติงานปัจจุบัน)
